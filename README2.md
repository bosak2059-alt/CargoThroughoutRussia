# main.py — Серверная часть (Python + pywebview)

## Отвечает за безопасность, работу с файловой системой, бизнес-логику и хранение данных.

| **Компонент** | **Технология** | **Назначение** |
|-----------|------------|------------|
|**load_data() / save_data()**|Функции чтения и записи JSON-файла (data.json)|Хранение всех заявок, транспорта, настроек и логов в одном портативном файле. Включает защиту от рекурсии при повреждении данных.|
|Класс **API**|Мост между интерфейсом и Python|Экспортируется в браузер через js_api=api. Предоставляет фронтенду все методы бизнес-логики без прямого доступа к файлам.|
|**get_orders()**, **get_vehicles()**, **get_settings()**, **get_logs()**|Методы выборки данных|Безопасная передача состояний из JSON в интерфейс для отрисовки таблиц и настроек.|
|**get_stats()**|Агрегатор статистики|Подсчёт общего количества заявок, суммарного веса груза и распределения по статусам (новый, в пути, доставлен) для дашборда.|
|**add_order()**|Создание заявки|Генерация ID и даты, проверка веса груза на соответствие грузоподъёмности выбранного ТС, сохранение и логирование.|
|**update_order()**|Редактирование заявки|Частичное обновление полей, повторная валидация веса при смене ТС или массы, логирование изменённых полей.|
|**delete_order()**|Редактирование заявки|Частичное обновление полей, повторная валидация веса при смене ТС или массы, логирование изменённых полей.|
|**add_vehicle()** / **delete_vehicle()**|CRUD для автопарка|Добавление нового ТС с авто-присвоением ID и удаление без нарушения связей в существующих заявках.|
|**calculate_time()**|Калькулятор времени доставки|Расчёт ориентировочного времени в пути: расстояние / avg_speed транспортного средства. Возвращает дни и часы.|
|**calculate_cost()**|Калькулятор стоимости|Умножение расстояния на тариф (tariff_per_km из настроек). Используется для быстрого расчёта перед созданием заявки.|
|**update_theme()** / **update_tariff()**|Управление настройками|Сохранение выбранной темы оформления и актуального тарифа в data.json без перезапуска приложения.|
|**create_backup()** / **list_backups()** / **restore_backup()**|Система резервного копирования|Создание снапшотов data.json в папку backups/, вывод списка доступных копий и восстановление данных из выбранного файла.|
|**export_csv()** / **import_orders()**|Модуль обмена данными|Генерация CSV-строки для Excel и парсинг входящих CSV/JSON-файлов. Автоматически пересчитывает ID при импорте.|
|**save_csv_export()** / **save_pdf_export()**|Сохранение экспортных файлов|Приём base64-данных от фронтенда и запись файлов в папку saves_exports/ без вызова системных диалогов ОС.|
|**if __name__ == "__main__"**:|Точка входа|Инициализация окна pywebview, загрузка index.html, привязка API и запуск цикла обработки событий.|

# app.js — Клиентская часть (JavaScript)

## Отвечает за реактивный интерфейс, валидацию на стороне клиента, визуализацию и работу с DOM.

| **Компонент** | **Технология** | **Назначение** |
|---------------|----------------|----------------|
|**loadAll()**|Центральный инициализатор|Асинхронный запрос всех данных у Python, кэширование в JS-переменные (currentOrders, currentVehicles) и запуск рендеринга всех модулей.|
|**initTheme()**|Менеджер темы|Считывание сохранённой темы из настроек, применение CSS-атрибута data-theme, реализация переключателя светлая/тёмная.|
|**setupTabs()**|Переключатель вкладок|Управление видимостью разделов (Заявки, Транспорт, Журнал, Настройки) через добавление/удаление CSS-класса active.|
|**setupForm() / setupVehicleForm() / setupEdit()**|Обработчики форм|Сбор данных из полей, отправка в Python через pywebview.api, очистка полей после успеха, вызов showToast() для уведомлений.|
|**renderVehicleSelect() / renderEditVehicleSelect()**|Генераторы <select>|Динамическое заполнение выпадающих списков ТС в форме создания и модальном окне редактирования.|
|**applyFilters() / setupFilters()**|Движок фильтрации|Фильтрация массива заявок по ключевому слову, статусу и диапазону дат. Обновляет таблицу в реальном времени без перезагрузки страницы.|
|**renderTable() / renderVehiclesTable() / renderLogs()**|Генераторы таблиц|Динамическое создание HTML-строк, подстановка статусов с цветовой индикацией, экранирование ввода через escapeHtml() (защита от XSS).|
|**setupCalc() / updateDeliveryTime()**|Калькулятор доставки|Запрос у Python стоимости и времени по дистанции и выбранному ТС. Автоматическая подстановка рассчитанной стоимости в поле заявки.|
|**renderStats() / renderChart()**|Визуализация дашборда|Обновление счётчиков (всего заявок, общий вес) и построение круговой диаграммы (Chart.js) по статусам для анализа загрузки.|
|**setupSettings() / loadBackups()**|Панель администратора|Интерфейс изменения тарифа, кнопки создания/восстановления бэкапов, загрузка списка резервных копий в <select>.|
|**setupImportExport()**|Импорт/Экспорт данных|Кнопка экспорта в CSV, выбор файла CSV/JSON для импорта, парсинг содержимого, передача в Python, обновление интерфейса.|
|**setupPDF()**|Генератор накладных|Загрузка кириллического шрифта Roboto, формирование PDF через jsPDF + autoTable, кодирование в base64 и отправка Python для сохранения.|
|**setupPrint()**|Печать интерфейса|Вызов window.print() для быстрого вывода текущего экрана на принтер (через системный диалог печати).|
|**showToast()**|Система уведомлений|Создание всплывающих сообщений (успех/ошибка/инфо) с автозакрытием через 3 секунды и анимацией исчезновения.|
|**escapeHtml()**|Утилита безопасности|Преобразование пользовательского ввода в безопасные HTML-сущности для предотвращения XSS-атак при рендеринге таблиц.|
|**changeStatus() / deleteOrder()**|Быстрые действия|Изменение статуса заявки из таблицы и удаление с подтверждением (confirm()), последующий вызов loadAll() для обновления данных.|

# Взаимодействие компонентов (Архитектура проекта)

Проект построен по гибридной клиент-серверной модели внутри одного исполняемого процесса:

1) Python (**main.py**) выступает как локальный бэкенд. Он управляет файлами, валидирует бизнес-правила (например, запрет перегруза ТС) и отдаёт данные по запросу.

2) JavaScript (**app.js**) работает в изолированном браузере **pywebview**. Он не имеет прямого доступа к диску, а общается с Python через асинхронные вызовы **window.pywebview.api.метод()**.

3) Хранение данных реализовано через единый **data.json**. Это упрощает деплой (не требует установки СУБД), позволяет легко переносить данные и создавать бэкапы.

4) Экспорт PDF/CSV реализован по схеме: **JS генерирует → кодирует в base64 → передаёт Python → Python сохраняет в локальную папку.** Это обходит ограничения браузера на автоматическое сохранение файлов без диалогов.
