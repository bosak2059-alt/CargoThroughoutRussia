# LogiFlow — Система управления логистикой

Гибридное десктоп-приложение для автоматизации работы диспетчеров и логистов. Позволяет вести учёт заявок, управлять автопарком, рассчитывать стоимость и время доставки, формировать печатные документы и отчёты. Приложение работает **полностью офлайн**, не требует внешних серверов, баз данных или подключения к интернету.

> 🎓 Проект разработан в рамках производственной практики. Архитектура и код соответствуют требованиям учебного заведения: разделение ответственности, безопасный IPC-мост, автономная работа, документированная сборка и модульность.

## 🛠 Стек технологий

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| **Backend** | Python 3.9+ | Бизнес-логика, работа с файлами, IPC-мост |
| **GUI-оболочка** | `pywebview` ≥4.4 | Встроенный браузерный движок ОС (WebView2/WebKit) |
| **Frontend** | HTML5 / CSS3 / Vanilla JS (ES6+) | Интерфейс, валидация, динамическая отрисовка |
| **Визуализация** | `Chart.js` (UMD) | Графики статистики и аналитики |
| **Документы** | `jsPDF` + `autoTable` | Генерация PDF-накладных на клиенте |
| **Хранение** | Локальный JSON (`data.json`) | Заявки, настройки, автопарк, журнал действий |
| **Сборка** | `PyInstaller` + `Pillow` | Упаковка в автономный `.exe` без зависимостей |

---

## 🏗 Архитектура приложения

Приложение построено по **гибридной модели**:
- **Python** выступает главным процессом: управляет жизненным циклом окна, читает/записывает данные, выполняет расчёты и резервное копирование.
- **WebView** отображает интерфейс на стандартных веб-технологиях.
- **Связь** осуществляется через асинхронный мост `window.pywebview.api`, что исключает прямой доступ фронтенда к файловой системе и повышает безопасность.
- **Данные** хранятся в структурированном JSON-файле. Модуль бэкапов автоматически создаёт снимки состояния при ручном запросе, а журнал действий логирует все CRUD-операции.

## 🚀 Установка и запуск

### Предварительные требования
- Windows 10/11 (встроенный WebView2 уже установлен)
- Python 3.9 или выше (`python --version`)
- Установленный `pip`

### Режим разработки
```bash
# 1. Клонирование/распаковка проекта
cd logistics_app

# 2. Установка зависимостей
pip install -r requirements.txt

# 3. Запуск приложения
python main.py
```

## Сборка в автономный .exe

### Ручная сборка

```
pyinstaller --onefile --windowed --name LogisticsApp --add-data "frontend;frontend" --icon logo.png main.py
```

| Флаг | Описание | 
|------|----------|
|--onefile|Упаковка всех ресурсов в один исполняемый файл|
|--windowed|Скрытие консольного окна при запуске|
|--add-data|Внедрение папки frontend внутрь сборки|
|--icon|Иконка приложения (PyInstaller автоматически конвертирует PNG в ICO)|
> Результат появится в папке dist/LogisticsApp.exe. Запускается на любом ПК без установки Python.

### Автоматическая сборка

Дважды кликните по build.bat → скрипт проверит зависимости, запустит PyInstaller и откроет папку с готовым файлом.
